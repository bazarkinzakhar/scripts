# bash-backup-tool

Универсальный скрипт для локального бэкапа директорий с поддержкой ротации, выгрузки в S3 и генерацией метрик для Prometheus. 

## Особенности
* **атомарная блокировка:** защита от запуска двух копий скрипта одновременно через `mkdir`
* **безопасная очистка:** ротация старых копий с защитой от пустых путей (`rm -rf /` исключен)
* **умный tar:** обработка кодов возврата (не падает, если файлы изменились в процессе чтения)
* **мониторинг:** создает `.prom` файл для `node_exporter` (textfile collector)
* **s3 integration:** опциональная загрузка в любое S3-совместимое хранилище

## Требования
* **OS:** Linux / macOS (BSD)
* **Зависимости:** `tar`, `find`, `df`, `awk`, `aws-cli` (только если нужен S3)

## использование и опции

### параметры запуска (cli)
| флаг | описание |
| :--- | :--- |
| `-s` | путь к исходным данным |
| `-d` | путь для сохранения архивов |
| `-r` | срок хранения локальных копий (в днях) |
| `-b` | адрес s3 бакета (напр. s3://bucket-name/path) |
| `-m` | путь для сохранения .prom метрик |
| `-h` | справка |

### переменные внутри скрипта
можно захардкодить значения в начале файла, чтобы не передавать флаги:
* `BACKUP_SRC` — источник бэкапа
* `BACKUP_DEST` — папка назначения
* `RETENTION_DAYS` — срок ротации (дефолт 7)
* `MIN_SPACE_KB` — минимальный порог места на диске (дефолт 1гб)
* `S3_BUCKET` — бакет для синхронизации
* `PROM_METRICS_DIR` — путь для node_exporter textfile collector

### подготовка к работе
* `chmod +x backup.sh` — дать права на исполнение
* `tar`, `find`, `df`, `awk` — наличие этих утилит в системе
* `aws configure` — настройка конфига и прав доступа, если нужен s3
* доступ на запись в `PROM_METRICS_DIR` для юзера, запускающего скрипт

### примеры команд
```bash
# локально с ротацией 14 дней
./backup.sh -s /etc/nginx -d /backups/nginx -r 14

# с выгрузкой в облако и метриками
./backup.sh -s /data -d /tmp/bckp -b s3://my-infra-storage -m /var/lib/node_exporter/textfile_collector
